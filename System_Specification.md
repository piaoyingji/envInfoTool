# 机构环境管理系统需求与设计说明文档

## 1. 系统概述
本系统旨在管理和检索各机构的不同环境（如开发测试环境、生产环境等）的登录信息。系统采用前后端分离的轻量化架构思想，通过两个主要的 HTML 页面来实现环境信息的录入和查询，底层数据依赖 CSV 文件进行存储和读取。

## 2. 系统功能与页面构成

### 2.1 管理员画面 (admin.html)
* **主要用途：** 供管理员用户输入和维护相关机构及其对应环境的详细信息。
* **画面构成：**
  * **数据录入表单：** 包含五个输入项，分别为：机构名、构筑环境名、URL、登录ID、密码。
  * **操作按钮：** 包含“保存/提交”按钮。
* **业务逻辑说明：**
  * 用户在表单中填入详细信息并提交。
  * **注意：** 考虑到纯浏览器端的 HTML/JS 无法直接覆写本地磁盘文件，管理端若要实现对 CSV 的直接保存，需要结合本地轻量级后端服务（如 Node.js / Python Flask 等）提供的 API 接口进行文件覆写，或者采用引导用户“下载生成最新 CSV 文件”的方式以更新源文件。

### 2.2 检索画面 (viewer.html)
* **主要用途：** 供普通用户通过简单的下拉选择，快速检索并展示特定机构下的所有环境及登录信息。
* **画面布局：**
  * **机构选择区：** 一个下拉列表组件 (Pulldown List / Select Box)，用于展示所有可供选择的机构名称。
  * **信息展示区：** 当用户在下拉列表中选择某一机构后，在此区域（可用表格(Table)或数据卡片(Card)的形式）显示该机构对应的所有环境明细。
* **业务逻辑说明：**
  * **数据加载与解析：** 页面加载时自动读取 CSV 文件列表并解析成数据对象。
  * **下拉列表去重：** 提取 CSV 文件中的第一列（机构名）数据，由于一个机构可能有多套环境（即第1列内容出现重复），系统会通过代码（如 JavaScript 的 `Set` 特性）进行去重处理，保证下拉列表里每个机构名只出现一次。
  * **数据联动展示：** 用户在下拉列表中选中一个机构后，触发选择事件（onChange）。系统根据当前选中的机构名，在全量数据中进行过滤匹配，将该机构名下对应的所有环境（环境名、URL、ID、PASSWORD）展示在下方的列表中。

## 3. 数据层设计 (CSV 数据字典)
数据源采用标准的 CSV 格式，按照顺序由以下 5 列组成：

| 列号 | 字段名称 | 说明 | 示例 |
| --- | --- | --- | --- |
| 1 | 机构名 (Organization) | 客户或机构的名称（同一机构存在多条时该字段会重复出现） | ABC科技有限公司 |
| 2 | 构筑环境名 (Environment) | 该机构名下分配的某个特定系统环境的名称 | 财务系统UAT测试环境 |
| 3 | URL | 该环境访问或登录的网络地址 | `https://uat.example.com/login` |
| 4 | 登录环境ID (Login ID) | 访问上述系统所需的管理员/测试用户登录账号 | `admin_test` |
| 5 | 登录环境PASSWORD (Password) | 访问上述系统所需的登录密码 | `Password123!` |

## 4. 技术实现重点与建议
* **CSV解析库：** 考虑到纯手写逗号分隔字符串解析可能遇到文本中内含逗号的问题，在后续代码实现中建议引入诸如 `PapaParse` 之类的成熟 JS 库来读取和解析 CSV 文件。
* **安全保护：** 环境的登录帐号密码等敏感信息目前以明文存于 CSV 内，在部署时应妥善存放该 CSV 文件，防止未授权用户直接通过 URL 路径下载该文件。可以结合服务器配置来进行目录权限拦截。
* **页面样式与用户体验：** 检索页面的交互应该保证平滑，并在未选择机构时提供友好的占位提示（如“请从下拉菜单中选择一个机构来查看详情”）。对于复制功能，可以考虑增加“一键复制密码”的小按钮以提高操作便利性。
